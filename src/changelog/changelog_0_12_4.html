0.12.4 branch overall additions and fixes:

  • Major changes
    • New Backend Help function that analyzes your setup
    • Extensive Email Functionality
    • New Invoice-based system of handling Registrations
    • AEC Subscription Process can be parted from any other registration process on your site (apart from also being able to show plans first or second during registration with joomla or CB)
    • This also means that we support every other component that uses the joomla login routines
    • New payment processor API that now also supports: CCBill, Paycom, Alertpay, epsnetpay, Paysignet, eway and moneyproxy
    • Heartbeat function that will carry out cron-job like actions
    • PHP based selection of Payment Plans including an extensive Restrictions system
    • Discount Coupons
    • Full German translation
  • Integration changes
    • Micro Integration System to attach php code with parameters to a payment plan
    • Already features MicroIntegrations for: emails, htaccess, mysql queries, setting of docman or VirtueMart groups, restricting downloads in docman and remository, restricting of listings amount of Mosets Tree and many more
    • SEF compatibility reworked
  • Backend changes
    • Large parts of the Backend revised to be more useable and better looking
    • Better backend handling of users, extensive sorting features
    • new Eventlog that gives you information on what is happening in the system
  • Minor changes
    • Custom Plans Intro, Thanks, Cancel and NotAllowed Page
    • Custom Text on most pages
    • added possibility to provide a custom IPN URL when you are working with balanced servers (paypal has a bad habit of not understanding this)
    • Expiration Cushion Setting
    • More User-Control over invoices (ability to cancel&redo on Mysubscription and Pending page)
    • Terms Of Service feature on Confirmation of Registration/Subscription
    • MySubscription, Expired and Pending pages reworked to break less layouts
    • Lots of bugfixes and optimizations
